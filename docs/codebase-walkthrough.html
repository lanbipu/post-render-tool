Ch. III · Pipeline

One Import button, five stages.

点下橙色 Import 后，pipeline.run_import() 把 CSV 依次喂给 parser / lens / camera / sequence / validator。 每个阶段都是纯 Python + UE API 的薄包装，成败写进同一个 PipelineResult 不抛异常。

1
csv_parser
读取、验列、提帧、估算 FPS。输出 CsvDenseResult。
2
lens_file_builder
按焦距分组 → 归一化畸变 → 写入 .ulens 资产。
3
camera_builder
Spawn CineCameraActor，设 Filmback，挂 LensComponent。
4
sequence_builder
创建 LevelSequence，9 个 channel × N 帧关键帧。
5
validator
FOV 一致性 + 跳变检测，出 ValidationReport。
Disguise CSV Dense file │ ▼ csv_parser.parse_csv_dense() │ → CsvDenseResult { frames, sensor_width_mm, focal_length_range, detected_fps } │ ├──────────────────────────────────────────┐ ▼ ▼ lens_file_builder coordinate_transform 按焦距分组 → 归一化畸变参数 Designer(m, Y-up) → UE(cm, Z-up) → 写入 LensFile transform_position / _rotation │ │ ▼ ▼ camera_builder sequence_builder Spawn CineCameraActor 对每一帧写入 9 个通道关键帧 挂 LensComponent + 畸变开关 Loc(X/Y/Z) + Rot(P/Y/R) + Focal + Aperture + Focus │ └──────────────────────────────────┐ ▼ validator FOV 交叉校验 + 位移/旋转跳变检测 → ValidationReport
⁓
Errors as data
PipelineResult 有 success: bool + error_message。调用方只看字段，不需要 try/except。
♯
帧 cadence 保持
CSV 从第 100 帧起 → Sequence 从时间 0 起，但 gap 保留。seq_idx = frame_number - first_frame_number。
≠
有理数帧率
23.976 fps = 24000/1001，不是 float。UE 的 FrameRate 用分数防浮点累积误差。

Twelve files, twelve roles.

Ch. VI · Open studies

What remains uncertain.

这个工具最大的未完成工作不在代码里，而在默认坐标映射。 config.py 里那三行 POSITION_MAPPING 和三行 ROTATION_MAPPING 注释明确写着 "INITIAL GUESSES"——它们是按 Disguise 的 Y-up 米制推断到 UE 的 Z-up 厘米的理论值， 但 stage 校准、舞台原点偏移、导演镜像需求都可能让真实值偏 180°。 Section IV 的 Spawn Test Camera 和 Section V 的 Apply / Save 正是为此存在： 用 UE viewport 自己当真值表，而不是让工程师在脑里做矩阵乘法。

§
  • sequence_builder.py:165  Transform channel 索引 [3/4/5] = Roll/Pitch/Yaw 硬编码，UE 版本更新可能静默错位。
  • lens_file_builder.py:93  焦距分组 O(n·m)；变焦镜头下可换 bisect 降到 O(n·log m)。
  • ui_interface.py:405  save_axis_mapping 的 importlib.reload 与 Apply 的内存写入存在短暂状态竞争。
  • csv_parser.py:233  int(float(row["frame"])) 双层转换是为 "100.0" 字符串准备的，未注释。
  • Figma 静态文本  不在契约中、无 CI 检查——POSITION (m → cm) 打反不会报错。
  • 坐标映射默认值  未用真实 CSV 验证；工具在 Section IV 提供了验证闭环但未执行。
